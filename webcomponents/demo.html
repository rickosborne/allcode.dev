Counting with a while loop

There's nothing magical about the name of the variable here. Instead of count, you could use fingers or place or whatever makes the most sense to you. Variables The variable is initialized with the numeric value of 1. Variables The while keyword starts a loop block. A Boolean expression is evaluated at the start of each iteration (repeat) of the loop. Boolean Expressions Expression Evaluation The condition evaluates to true as long as the value of the count is less than or equal to the number 5. Boolean Expressions The loop body is a block which will get executed if the condition evaluates to true. This block is very simple and only does one thing. Blocks can do much more than this, and may do many things one after another. Blocks The loop body increments (adds 1 to) the count variable. Increment and Decrement The first time the condition is evaluated, what is the value of the count variable? 1 The count variable is initialized with a value of 1. This value is not modified before the loop begins. Variables When the condition eventually evaluates to false, what is the value of the count variable? 6 The first time through the loop, the count starts at 1 and is incremented to 2. Next time through, the count is modified from 2 to 3, then 3 to 4, and 4 to 5. Because the condition expression evaluates as true when the count is 5, the loop body runs a fifth time. In that fifth time through, the count is incremented from 5 to 6. When the condition is evaluated again, it is now false as 6 is not less than or equal to 5. Which parts of the code are repeated in the loop? The loop body, which is a block and could have many lines of code, will be executed each time the loop is repeated. Blocks How many times would the loop execute if you changed the expression from "less than or equal" to just "less than"? 4 The first time through the loop, the count starts at 1 and is incremented to 2. Next time through, the count is modified from 2 to 3, then 3 to 4, and 4 to 5. When the condition is evaluated, 5 is no longer “less than” 5, so the condition is false and the loop does not run the fifth time. What would happen if you did not increment the count variable inside the loop body? The count would never change, leading to an infinite loop. The count variable is initialized with the value 1. If it is never updated within the loop, that value of 1 will never change, and the expression “1 is less than or equal to 5” will always be true. Since the condition is always true, the loop will never end.
